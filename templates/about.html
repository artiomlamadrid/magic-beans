About Magic Beans

A comprehensive stock analysis platform for educational and research purposes

Project Overview

Magic Beans is a web-based stock analysis platform built with Flask that provides comprehensive financial analysis tools for educational and research purposes. The application leverages real-time data from Yahoo Finance to perform sophisticated valuation calculations and present them in an intuitive, user-friendly interface.

This project was developed as a final project for CS50's Introduction to Computer Science course, demonstrating advanced programming concepts including object-oriented design, web development, data analysis, and comprehensive testing methodologies.

Core Features

Real-Time Data Fetching

Direct integration with Yahoo Finance API to fetch live financial data including stock prices, financial statements, and market metrics.

Advanced Valuation Models

Implementation of multiple valuation methodologies including DCF, DDM, and P/E analysis with customizable parameters.

Data Management

Save and load functionality for offline analysis and comparison of historical data points.

Configurable Analysis

User-adjustable parameters for discount rates, growth assumptions, and projection periods to suit different analysis scenarios.

How It Works

1. Data Acquisition

Magic Beans uses the yfinance Python library to fetch real-time financial data from Yahoo Finance. The application can retrieve:

  • Basic Information: Current price, market cap, sector, industry details
  • Financial Statements: Balance sheets, income statements, cash flow statements
  • Market Data: Historical prices, dividends, stock splits
  • Analyst Data: Earnings estimates, recommendations, target prices

2. Data Storage & Management

The application provides flexible data management capabilities:

  • Save Function: Stores fetched data as JSON files in the saved_data/ directory for offline access
  • Load Function: Retrieves previously saved data for analysis without requiring new API calls
  • Export Function: Converts data to CSV format for external analysis tools

3. Financial Analysis Engine

The core analysis engine implements multiple valuation models:

Discounted Cash Flow (DCF) Model

Projects future free cash flows and discounts them to present value using the company's weighted average cost of capital (WACC).

  • Projection Period: 5-20 years (default: 12 years)
  • Fade Period: 3-15 years for growth normalization (default: 8 years)
  • Terminal Growth: 0-10% perpetual growth rate (default: 4%)
  • Discount Rate: Based on industry risk and company characteristics
Dividend Discount Model (DDM)

Values the company based on the present value of expected future dividend payments.

  • Dividend Growth Rate: 0-20% based on historical patterns (default: 4%)
  • Required Return: 3-15% investor return expectation (default: 8%)
  • Projection Period: 3-10 years (default: 6 years)
Price-to-Earnings (P/E) Analysis

Compares the company's valuation to industry peers using earnings multiples.

  • Forward vs. Trailing P/E: User-selectable preference
  • Sector Premium: Adjustments for sector-specific characteristics
  • Peer Comparison: Industry average multiple analysis

Technical Architecture

Backend Components

  • Flask Application (app.py): Web framework handling HTTP requests and routing
  • Stock Class (stock.py): Data fetching and management using yfinance API
  • StockAnalysis Class (stock_analysis.py): Core valuation algorithms and calculations
  • Analysis Services (analysis_service.py): Business logic layer with formatting and orchestration
  • Helper Functions (helpers.py): Utility functions for data processing

Frontend Components

  • Bootstrap 5: Responsive CSS framework for modern UI design
  • Jinja2 Templates: Server-side rendering with minimal client-side logic
  • Interactive Forms: Parameter configuration with real-time validation
  • Data Visualization: Clean tabular presentation of financial data

Testing Infrastructure

  • Unit Tests: 153 comprehensive tests covering all major functionality
  • Mock Testing: Isolated testing of components without external API dependencies
  • Error Handling: Extensive testing of edge cases and failure scenarios

Analyze comprehensive valuation results and investment insights
